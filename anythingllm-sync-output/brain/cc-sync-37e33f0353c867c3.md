# Sync Model - Incremental & Full

**Type:** fact
**Tier:** permanent
**Confidence:** 0.90
**Created:** Mon Feb 09 2026 15:30:38 GMT-0300 (Brasilia Standard Time)

---

The service employs both incremental (*/30min) and full (*/6h) sync schedules via cron.  Incremental syncs use `updated_at` cursors, while full syncs copy all data.  Batch upserts of 100 rows are used to respect Supabase API limits.
