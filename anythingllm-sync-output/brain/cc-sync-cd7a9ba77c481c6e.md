# Database Schema - Core Tables

The system utilizes a 6-table database schema: sops (SOP registry), sop_versions (version history), sop_executions (audit log), sop_conflicts (conflict detection), sop_extraction_queue (pending commands), and sop_templates (reusable patterns).
